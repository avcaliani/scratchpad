# 3. API

**Stack:** Python · FastAPI · Databricks SQL Connector · Pydantic  
**Purpose:** REST API — ACL between the frontend and the Databricks SQL Warehouse.

---

## Endpoints

### `GET /health`
Returns `200 OK`. No body required.

### `GET /api/trips/{trip_id}`

**Validation:** `trip_id` must be a 64-character lowercase hex string. Return `400` otherwise.

**Query:** Inline parameterized SELECT against `workspace.gold.nyctaxi_trips WHERE trip_id = %s`. Pass `trip_id` as the parameter value — do not interpolate it into the SQL string.

**Response (200):**

Trip fields:

| Field | Type |
|---|---|
| trip_id | string |
| tpep_pickup_datetime | string (ISO 8601) |
| tpep_dropoff_datetime | string (ISO 8601) |
| trip_distance | float |
| fare_amount | float |
| pickup_zip | int |
| dropoff_zip | int |
| ingestion_timestamp | string (ISO 8601) |

Quality flags are nested under `data_quality_checks`:

| Field | Type |
|---|---|
| status | string (`VALID` or `INVALID`) |
| invalid_fare | bool |
| invalid_distance | bool |
| invalid_timestamp | bool |
| null_critical_fields | bool |
| unrealistic_distance | bool |
| unrealistic_fare | bool |
| zero_distance_paid | bool |

The Pydantic model handles the transformation from the flat SQL result to this nested shape. The gold table schema remains flat.

**Error responses:**

| Code | Condition |
|---|---|
| `400` | `trip_id` fails format validation |
| `404` | No row found in gold table |
| `503` | Warehouse unreachable or query failure |

## Architecture

```
app.py              # FastAPI app, router registration, StaticFiles mount
api/
  models.py         # DataQualityChecks, TripResponse — both have from_row() classmethods
  db.py             # _QUERY constant, query_trip(), lazy singleton connection
  routes.py         # APIRouter, trip_id validation, structured JSON logging
```

## Warehouse Connectivity

Query is inlined in `api/db.py` using `%s` as the placeholder (DB-API 2.0). Never interpolate `trip_id` into the SQL string. A module-level `_state` dict holds a single persistent connection, lazy-initialized on first use and reset to `None` on any exception to trigger reconnect. Connection parameters are read from `DATABRICKS_HOST` and `DATABRICKS_HTTP_PATH` environment variables; credentials are inherited automatically by the Databricks Apps runtime.
